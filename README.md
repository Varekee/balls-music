# Balls Music — сайт на хостинге + музыка на вашем ПК

Схема: на хостинге лежит только `index.html`. Музыка остаётся на вашем компьютере.
Компьютер сам раздаёт её по сети через маленький локальный сервер, а Cloudflare Tunnel
делает этот сервер доступным из интернета — без проброса портов и настройки роутера.

## Что в наборе
- `index.html` — сайт-плеер (это единственный файл, который загружается на хостинг)
- `local_server.py` — сервер, который запускается на ПК и раздаёт музыку
- эта инструкция

## Шаг 1. Придумайте код доступа

Откройте `local_server.py`, найдите строку:
```python
ACCESS_KEY = "замените-меня-на-свой-код"
```
Замените на свой секретный код (любые буквы/цифры, без пробелов). Он нужен, чтобы
музыку не мог слушать кто угодно, кто случайно найдёт сайт — без правильного кода
сервер отвечает ошибкой 401 и ничего не отдаёт.

## Шаг 2. Подготовьте музыку на ПК
1. Создайте на компьютере папку, например `balls-music/`.
2. Положите туда `local_server.py` и папку `music/` со всей вашей музыкой
   (можно с подпапками — сервер обойдёт их рекурсивно).
   ```
   balls-music/
   ├── local_server.py
   └── music/
       ├── Artist - Song.mp3
       └── Album/
           └── Other Artist - Track.flac
   ```
3. Запустите сервер:
   ```
   python local_server.py
   ```
   Появится сообщение `Сервер музыки запущен: http://127.0.0.1:8080`.
   Список треков собирается на лету — просто добавляйте файлы в `music/`,
   ничего пересобирать вручную не нужно. Окно с сервером нужно держать открытым
   всё время, пока хотите слушать музыку через сайт.

## Шаг 3. Установите Cloudflare Tunnel (cloudflared)
Скачайте `cloudflared` с официальной страницы Cloudflare для вашей ОС:
https://developers.cloudflare.com/cloudflare-one/connections/connect-networks/downloads/

Затем в отдельном окне терминала (сервер из шага 1 пусть продолжает работать) запустите:
```
cloudflared tunnel --url http://localhost:8080
```
В выводе появится строка вида:
```
https://random-words-1234.trycloudflare.com
```
Это и есть публичный адрес вашего домашнего сервера музыки. Скопируйте его.

Важно: такой быстрый туннель бесплатный и не требует ни аккаунта, ни домена, но
адрес меняется при каждом перезапуске `cloudflared`. Если вас это не устраивает
и вы хотите один и тот же адрес навсегда — можно завести бесплатный аккаунт
Cloudflare и подключить к нему домен, чтобы сделать именованный туннель со
стабильным адресом. Напишите мне, если захотите так сделать — распишу шаги.

## Шаг 4. Укажите адрес и код доступа в сайте
1. Откройте `index.html` в любом текстовом редакторе.
2. Найдите строку:
   ```js
   const MUSIC_SERVER = 'https://ВАШ-АДРЕС.trycloudflare.com';
   ```
3. Замените на адрес из шага 3 (без слэша на конце), сохраните файл.
4. Код доступа отдельно прописывать в `index.html` не нужно — на сайте появится
   экран с полем ввода, куда вы (и те, кому дадите доступ) будете вводить код
   из `local_server.py` при каждом открытии сайта.

## Шаг 5. Загрузите сайт на GitHub Pages
1. Зайдите в репозиторий `balls-music` на GitHub.
2. **Add file → Upload files** → перетащите обновлённый `index.html` (он заменит старый) → **Commit changes**.
3. Подождите минуту — сайт обновится по тому же адресу `https://ваш-логин.github.io/balls-music/`.

## Шаг 6. Дальнейшее использование
Каждый раз, когда хотите послушать музыку через сайт:
1. Запустите `local_server.py` на ПК.
2. Запустите `cloudflared tunnel --url http://localhost:8080`.
3. Если адрес туннеля изменился — обновите `MUSIC_SERVER` в `index.html` и загрузите новую версию на GitHub.
4. Откройте сайт, введите код доступа — плеер подтянет список треков.

## Приватность
- Настоящая защита музыки — код доступа (`ACCESS_KEY`) в `local_server.py`: без него сервер
  отвечает ошибкой 401 и не отдаёт ни список треков, ни сами файлы, даже если кто-то узнает
  адрес туннеля.
- Страница сайта помечена как `noindex` — поисковики её не индексируют, но по прямой ссылке
  она всё равно открывается: GitHub Pages на бесплатном тарифе не умеет делать страницу полностью
  приватной, даже если сам репозиторий private. Экран с кодом доступа — это защита именно
  на уровне приложения, встроенная в сам сайт.
- Код доступа и адрес туннеля сообщайте только тем, кому лично разрешаете слушать музыку —
  это и делает доступ ограниченным узким кругом лиц, а не публичной раздачей.
- Если захотите более строгую защиту (полноценный логин по email через Cloudflare Access) —
  это возможно, но потребует свой домен, привязанный к бесплатному аккаунту Cloudflare.
  Скажите — распишу отдельно.
